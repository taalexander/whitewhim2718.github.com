Moving to Jekyll

Hello and welcome to my new blog. I’ve decided to move from my custom blog built with Django over to a Jekyll based blog for a variety of reasons. First and foremost Jekyll is more extensible and has many pre-made custom plugins that allow me to add a variety of features to my blog without the need for me to write them myself. As I’m so busy these days this is a must if I’m every actually to get around to blogging.

Secondly

Jekyll generates a static site. This allows me to host my site at Github Pages free of charge. More importantly this will allow me to take down the server that was previously used for hosting my old blog and repurpose it. Additionally the static nature of Jekyll allows the blog to remain super fast even under heavy load as the HTML, CSS and JS can be hosted by CDN and there is no server side processing to bottleneck serving requests.

Octopress

On top of Jekyll I am using Octopress to build the blog. Octopress provides a ton of features that are not included in barebones Jekyll. Octopress gives has custom Rake scripts that allow easy blog management and deployment. Additionally custom blog features are built in such as archives, comments and RSS. This makes the design and setup of a static blog a snap.

Theming

Finally I want to give a shout-out to Pankaj Kumar in India for the use of his Fabric theme. It’s clean and responsibe and exactly what I was looking for. I don’t have a stylish bone in my body so I’m lucky he was thoughtful enough to open-source his theme.
